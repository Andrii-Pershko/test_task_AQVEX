# AQVEX — каталог товарів

Односторінковий каталог товарів AQVEX на **React**, **TypeScript**, **Vite** з **Redux Toolkit**, **React Router** та **Tailwind CSS**. Дані підвантажуються з публічного API.

## Функціонал

- **Головна сторінка** — сітка карток товарів (ціна, знижка, рейтинг, відгуки, категорія, наявність, вибір обʼєму).
- **Пошук** — фільтрація за назвою товару в реальному часі.
- **Сортування** — «По популярности», «От дешевших», «От дорогих» (рейтинг/відгуки та ціна).
- **Пагінація** — 12 товарів на сторінку; номери сторінок з трьома крапками (1, 2, 3, 4, …, 6 тощо).
- **Картка товару** — зображення з заглушкою під час завантаження; при помилці/404 — скелет-заглушка.
- **Випадаюче меню обʼємів** — якщо обʼємів більше одного, показується вибір обʼєму (у стилі фільтрів); за замовчуванням обраний перший.
- **Сторінка товару** (`/product/:id`) — відкривається по кліку на картку; повна картка товару та блок «Опис товару» з деталями (категорія, рейтинг, наявність, доступні обʼєми).
- **Layout** — спільний хедер з логотипом і навігацією, футер; роутинг винесено в `App`, layout — в окремий компонент.
- **Стилі** — Tailwind, CSS-змінні (кольори, шрифти Klein та Afacad), однаковий вигляд на головній та на сторінці товару.
- **Деплой** — збірка готова до публікації на GitHub Pages (`base: './'` у Vite).

## Стек

- **React 18** + **TypeScript**
- **Vite 5**
- **Redux Toolkit** — стан і запит до API (`/api/v1/products`)
- **React Router 6** — головна та сторінка товару
- **Tailwind CSS** — стилі
- **Axios** — HTTP-запити
- **vite-plugin-svgr** — імпорт SVG як React-компонентів

## Як запустити

```bash
npm install
npm run dev
```

Додаток відкриється на `http://localhost:5173` (або на порту, який покаже Vite).

## Скрипти

| Команда | Опис |
|--------|------|
| `npm run dev` | Запуск dev-сервера |
| `npm run build` | Production-збірка |
| `npm run preview` | Перегляд збірки локально |
| `npm run deploy` | Збірка та публікація на GitHub Pages (гілка `gh-pages`) |

## Деплой на GitHub Pages

1. У `vite.config.ts` для репозиторію не в корені вкажи `base: '/назва-репо/'`.
2. Ініціалізуй репозиторій, додай remote на GitHub.
3. Виконай `npm run deploy` — зʼявиться/оновиться гілка `gh-pages`; у налаштуваннях репозиторію ввімкни публікацію з цієї гілки.

## Структура проєкту (коротко)

- `src/pages/` — головна та сторінка товару.
- `src/components/` — Home, Search, SortFilter, Paginations, ProductCart, Container, Layout, Footer тощо.
- `src/store/` — Redux store та `productsSlice` (запит до API, список товарів).
- `src/assets/` — шрифти, зображення, SVG.
- `public/` — статика (наприклад, favicon).
